Tentang Proyek : Platform E-Commerce UMKM Lokal adalah aplikasi berbasis web yang bertujuan membantu UMKM memasarkan produknya secara digital dengan komisi rendah, fitur social commerce, dan integrasi pembayaran lokal. Sistem ini dirancang agar mudah digunakan oleh pelaku UMKM yang belum terbiasa dengan teknologi, sekaligus memberikan kontrol penuh atas data pelanggan dan dukungan logistik lokal untuk efisiensi operasional.

1. Tahap Inisiasi Proyek

1.1
Identifikasi Masalah

Masalah yang dihadapi oleh Dinas Koperasi dan UMKM serta pelaku usaha lokal adalah sebagai berikut:

  • Biaya komisi di marketplace besar terlalu tinggi (15–20%) sehingga mengurangi keuntungan UMKM.
  • Brand UMKM sulit berkembang karena tertutupi oleh promosi dari penjual besar di marketplace.
  • UMKM tidak memiliki kontrol terhadap data pelanggan dan perilaku pembeli.
  • Ongkos kirim lokal relatif mahal karena tidak ada sistem logistik khusus wilayah tertentu.
  • Tingkat literasi digital pelaku UMKM masih rendah sehingga sulit beradaptasi dengan platform online kompleks.
  • Tidak adanya integrasi sistem pembayaran yang memudahkan transaksi (e-wallet, transfer bank, COD).
  • Kurangnya sistem yang membantu promosi berbasis komunitas seperti social commerce (live streaming, afiliasi, dan ulasan produk).

1.2
Tujuan Proyek

Tujuan utama proyek Platform E-Commerce UMKM Lokal ini adalah:

  • Membuat platform e-commerce dengan biaya komisi rendah agar UMKM dapat meningkatkan keuntungan.
  • Menyediakan fitur social commerce (live streaming, afiliasi, dan review) untuk meningkatkan interaksi antara penjual dan pembeli.
  • Menghadirkan sistem pembayaran terintegrasi dengan berbagai metode (e-wallet, transfer, COD).
  • Mengembangkan dashboard manajemen toko bagi UMKM agar mudah mengelola produk, pesanan, dan keuangan.
  • Menyediakan fitur rekomendasi berbasis AI untuk membantu pembeli menemukan produk lokal yang relevan.
  • Mengimplementasikan logistik lokal terintegrasi untuk menekan biaya pengiriman dan mempercepat distribusi.
  • Memberikan pelatihan dan panduan digital agar UMKM yang belum terbiasa teknologi bisa beradaptasi.

1.3
Ruang Lingkup Proyek

Termasuk dalam proyek

  • Pengembangan website e-commerce berbasis web responsif untuk desktop & mobile.
  • Modul toko online UMKM (registrasi, manajemen produk, stok, dan pesanan).
  • Fitur shopping cart dan checkout dengan integrasi ke payment gateway.
  • Dashboard penjual (analytics, laporan penjualan, pengaturan toko).
  • Sistem logistik lokal dengan pelacakan pengiriman sederhana.
  • Fitur review & rating produk dan penjual.
  • Fitur social commerce: live streaming dan sistem afiliasi.
  • Sistem komisi otomatis dan settlement ke rekening UMKM.
  • Sistem rekomendasi berbasis AI (produk terkait & tren lokal).
  • Panel admin untuk pengelolaan platform (pengguna, transaksi, dan laporan).

Tidak Termasuk dalam proyek

  • Aplikasi mobile native (Android/iOS) — hanya versi web responsif.
  • Integrasi dengan marketplace besar (Shopee, Tokopedia, dll).
  • Layanan logistik nasional (hanya fokus pada jaringan lokal).
  • Fitur pinjaman modal atau integrasi fintech pembiayaan.
  • Pengembangan sistem pelatihan terpisah (akan disediakan melalui modul edukasi di platform).

3. Analisis Kebutuhan Fungsional

3.1
Modul Manajemen Pengguna

  • FR-001 : Sistem harus dapat melakukan registrasi akun UMKM, pembeli, dan admin dengan verifikasi email atau nomor HP.
  • FR-002 : Sistem harus menyediakan fitur login dengan username/email dan password.
  • FR-003 : Sistem harus memungkinkan pengguna mengedit profil (nama, alamat, foto, informasi kontak).
  • FR-004 : Sistem harus memiliki role-based access control: Admin, Penjual (UMKM), Pembeli, Kurir.
  • FR-005 : Sistem harus menyediakan fitur lupa password (reset via email atau OTP).

3.2
Modul Manajemen Toko & Produk (Seller Dashboard)

  • FR-006 : Penjual dapat membuat dan mengelola toko online-nya sendiri.
  • FR-007 : Penjual dapat menambah, mengedit, dan menghapus produk dengan informasi lengkap (nama, deskripsi, harga, stok, gambar, kategori).
  • FR-008 : Sistem harus menampilkan status ketersediaan produk secara real-time.
  • FR-009 : Sistem menyediakan fitur upload massal produk menggunakan file CSV/Excel.
  • FR-010 : Penjual dapat melihat statistik penjualan, pengunjung toko, dan rating produk di dashboard.
  • FR-011 : Penjual dapat membuat promosi, voucher, dan diskon untuk produk tertentu.

3.3
Modul Katalog & Pencarian Produk

  • FR-012 : Sistem menampilkan katalog produk dari seluruh UMKM yang terdaftar.
  • FR-013 : Sistem menyediakan pencarian produk berdasarkan kata kunci, kategori, harga, lokasi, atau toko.
  • FR-014 : Sistem harus mendukung filter (harga, rating, kategori) dan urutan hasil pencarian (terbaru, termurah, terlaris).
  • FR-015 : Sistem harus menampilkan detail produk lengkap (gambar, deskripsi, stok, ulasan, rating, informasi toko).

3.4
Modul Keranjang Belanja & Pemesanan

  • FR-016 : Pembeli dapat menambah produk ke keranjang dan mengubah jumlah pembelian.
  • FR-017 : Pembeli dapat melanjutkan ke proses checkout dan memilih metode pembayaran.
  • FR-018 : Sistem harus menghitung total biaya (subtotal, ongkir, diskon, dan pajak).
  • FR-019 : Sistem harus mendukung multi-toko checkout (pembeli bisa beli dari beberapa UMKM sekaligus).
  • FR-020 : Sistem menyimpan riwayat pesanan pembeli.

3.5
Modul Pembayaran & Komisi

  • FR-021 : Sistem harus terintegrasi dengan berbagai payment gateway (QRIS, e-wallet, transfer bank, COD).
  • FR-022 : Sistem harus mencatat status transaksi (menunggu, berhasil, gagal, refund).
  • FR-023 : Sistem menghitung dan memotong komisi platform secara otomatis dari setiap transaksi.
  • FR-024 : Sistem harus melakukan proses settlement ke rekening UMKM secara otomatis sesuai jadwal.

3.6
Modul Logistik & Pengiriman

  • FR-025 : Sistem harus terhubung dengan jasa logistik lokal.
  • FR-026 : Pembeli dapat memilih opsi pengiriman (reguler, cepat, ambil di toko).
  • FR-027 : Kurir dapat melihat daftar pesanan yang perlu dikirim.
  • FR-028 : Sistem menampilkan status pengiriman secara real-time (tracking).
  • FR-029 : Sistem harus mengirim notifikasi ke pembeli ketika pesanan dikirim atau diterima.

3.7
Modul Review & Rating

  • FR-030 : Pembeli dapat memberikan ulasan dan rating untuk produk yang telah dibeli.
  • FR-031 : Sistem menampilkan rata-rata rating produk dan komentar pembeli.
  • FR-032 : Admin dapat memoderasi atau menghapus ulasan yang melanggar aturan.

3.8
Modul Social Commerce & Affiliate

  • FR-033 : Penjual dapat melakukan live streaming promosi produk langsung dari dashboard.
  • FR-034 : Pengguna dapat membagikan link afiliasi produk dan mendapatkan komisi dari penjualan.
  • FR-035 : Sistem menampilkan daftar top affiliate dan performa promosi sosial.

3.9
Modul Laporan & Analitik

  • FR-036 : Admin dapat melihat laporan penjualan, transaksi, dan komisi.
  • FR-037 : Dinas Koperasi dapat mengunduh laporan perkembangan UMKM (penjualan, omzet, aktivitas).
  • FR-038 : Sistem menampilkan grafik dan insight seperti produk terlaris, toko aktif, dan pertumbuhan penjualan.
  • FR-039 : Sistem dapat mengekspor laporan ke format PDF dan Excel.

4. Analisis Kebutuhan Fungsional

4.1
Performance (Kinerja)

  • NFR-001 : Waktu loading halaman utama maksimal 3 detik pada koneksi 5 Mbps.
  • NFR-002 : Sistem harus mampu menangani minimal 500 pengguna aktif bersamaan (concurrent users).
  • NFR-003 : Proses pencarian produk harus mengembalikan hasil dalam waktu ≤ 2 detik.
  • NFR-004 : Dashboard admin dan seller harus tetap responsif meskipun terdapat ribuan data transaksi.
  • NFR-005 : Sistem harus mendukung caching pada data produk populer untuk mempercepat akses.

4.2
Security (Keamanan)

  • NFR-006 : Semua komunikasi data harus menggunakan protokol HTTPS (SSL/TLS).
  • NFR-007 : Password pengguna harus dienkripsi menggunakan bcrypt atau algoritma setara.
  • NFR-008 : Implementasi Two-Factor Authentication (2FA) untuk akun admin dan seller.
  • NFR-009 : Sistem harus mencegah serangan SQL Injection, XSS, dan CSRF.
  • NFR-010 : Otentikasi menggunakan JWT (JSON Web Token) untuk menjaga keamanan sesi login.
  • NFR-011 : Session akan otomatis berakhir setelah 30 menit tidak aktif.
  • NFR-012 : Aktivitas penting (login, transaksi, perubahan data) harus dicatat dalam audit log.

4.3
Usability (Kemudahan Penggunaan)

  • NFR-013 : Tampilan antarmuka harus sederhana dan mudah dipahami oleh pengguna non-teknis (UMKM).
  • NFR-014 : Desain harus responsif dan dapat diakses dengan baik di perangkat mobile dan desktop.
  • NFR-015 : Tersedia fitur panduan interaktif atau tutorial video untuk membantu pengguna baru.
  • NFR-016 : Bahasa yang digunakan adalah Bahasa Indonesia yang komunikatif dan mudah dipahami.
  • NFR-017 : Navigasi utama (menu, ikon, tombol) harus konsisten di seluruh halaman.

4.4
Reliability (Keandalan)

  • NFR-018 : Sistem harus memiliki tingkat ketersediaan (uptime) minimal 99% per bulan.
  • NFR-019 : Sistem harus melakukan backup database otomatis setiap hari.
  • NFR-020 : Waktu pemulihan sistem maksimal 4 jam jika terjadi gangguan (recovery time ≤ 4 jam).
  • NFR-021 : Sistem harus tetap dapat beroperasi meskipun salah satu layanan (misalnya logistik) tidak aktif, menggunakan fallback mechanism.
  • NFR-022 : Sistem harus memiliki redundansi server untuk mencegah kehilangan data.

4.5
Maintainability (Pemeliharaan)

  • NFR-023 : Kode program harus ditulis dengan standar konvensi dan dokumentasi lengkap.
  • NFR-024 : Sistem harus dibangun menggunakan arsitektur modular dan terpisah (frontend-backend) untuk memudahkan pembaruan.
  • NFR-025 : API dan modul integrasi pihak ketiga (payment, logistik) harus memiliki dokumentasi teknis.
  • NFR-026 : Menggunakan version control system (Git) untuk mengelola pengembangan.
  • NFR-027 : Pengujian otomatis (unit test & integration test) wajib diterapkan sebelum deployment.

5. Spesifikasi Teknis

5.1
Teknologi yang Digunakan

Frontend : Next.js / React.js, Tailwind CSS, DaisyUI
Backend : Laravel
Database : MySQL
Payment Gateway : Midtrans
Logistik API : RajaOngkir
AI Rekomendasi : Python (FastAPI + Scikit-learn)
Web Server : Nginx
Hosting : AWS
Version Control : GitHub
Testing Tools : Postman

5.2
Kebutuhan Server & Client

  • Server : CPU 4 Core, RAM 8 GB, Storage 100 GB SSD, OS Ubuntu 22.04
  • Client: : Browser modern (Chrome/Firefox/Safari), koneksi internet ≥ 1 Mbps, responsif di mobile & desktop.

5.3
Keamanan & Pemeliharaan

  • SSL (HTTPS) di seluruh domain
  • Enkripsi password (bcrypt)
  • Backup otomatis harian
  • Otentikasi JWT
  • CI/CD Deployment (GitHub Actions / Docker)
