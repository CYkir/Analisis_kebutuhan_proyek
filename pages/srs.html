SOFTWARE REQUIREMENT SPECIFICATION (SRS)

Platform E-Commerce UMKM Lokal

Versi 1.0

1. PENDAHULUAN

1.1 Tujuan Dokumen

Dokumen ini menjelaskan spesifikasi kebutuhan sistem untuk Platform E-Commerce UMKM Lokal.

Dokumen ini ditujukan untuk tim pengembang, stakeholder, dan pihak terkait yang terlibat dalam pengembangan serta implementasi sistem.

1.2 Ruang Lingkup Sistem

Platform E-Commerce UMKM Lokal adalah aplikasi berbasis web yang dirancang untuk membantu pelaku UMKM menjual produk mereka secara online kepada konsumen lokal.

Sistem ini mendukung fitur transaksi digital, integrasi pembayaran, manajemen produk, pengiriman, serta dashboard admin untuk monitoring.

2. DESKRIPSI UMUM

2.1 Perspektif Produk

Sistem ini merupakan aplikasi web yang akan menjadi platform penghubung antara penjual UMKM, pembeli, kurir, bank/payment gateway, dan admin Dinas Koperasi.

Aplikasi akan terintegrasi dengan layanan pembayaran digital dan sistem pelacakan pengiriman.

2.2 Fungsi Produk

  • Manajemen akun pengguna (UMKM, Pembeli, Admin, Kurir)
  • Manajemen produk UMKM
  • Transaksi online dan integrasi pembayaran
  • Pelacakan pesanan dan status pengiriman
  • Dashboard analitik untuk admin dan UMKM
  • Fitur ulasan dan rating produk

2.3 Karakteristik Pengguna

Tipe Pengguna Karakteristik Tingkat Keahlian
Penjual UMKM Menjual produk, mengelola toko Basic–Intermediate
Pembeli Membeli produk, melakukan pembayaran Basic
Admin Platform Mengelola data dan memonitor sistem Advanced
Kurir Melacak dan mengupdate status pengiriman Basic
Dinas Koperasi Melihat laporan dan statistik UMKM Intermediate

3. Kebutuhan Fungsional

3.1 Modul Manajemen Pengguna

- FR-001: Registrasi Pengguna

  • Deskripsi: Sistem menyediakan form registrasi untuk UMKM dan pembeli.
  • Input: Nama, email, password, nomor HP.
  • Proses: Validasi data dan verifikasi email.
  • Output: Akun aktif dan terdaftar.
  • Prioritas: High

- FR-002: Login & Autentikasi

  • Input: Email dan password.
  • Proses: Validasi kredensial, token session (JWT).
  • Output: Redirect ke dashboard sesuai role.
  • Prioritas: High

3.2 Modul Produk dan Katalog

- FR-010: Manajemen Produk

  • Deskripsi: Penjual dapat menambah, mengedit, dan menghapus produk.
  • Input: Nama produk, harga, deskripsi, foto, stok.
  • Proses: CRUD operation, validasi gambar.
  • Output: Produk tersimpan dan tampil di katalog.
  • Prioritas: High

- FR-011: Pencarian & Filter Produk

  • Input: Kata kunci (nama, kategori, harga).
  • Proses: Query database dengan filter.
  • Output: Daftar produk sesuai kriteria.
  • Prioritas: High

3.3 Modul Transaksi dan Pembayaran

- FR-020: Checkout & Pembayaran Online

  • Deskripsi: Pembeli dapat melakukan checkout dan pembayaran.
  • Input: Produk, alamat, metode pembayaran.
  • Proses: Validasi data, integrasi payment gateway.
  • Output: Status “Menunggu Pembayaran”.
  • Prioritas: High

- FR-021: Konfirmasi & Status Pesanan

  • Proses: Update status pesanan dari penjual dan kurir.
  • Output: Status real-time (Diproses, Dikirim, Selesai).
  • Prioritas: High

3.4 Modul Admin & Laporan

- FR-030: Manajemen Pengguna & UMKM

  • Deskripsi: Admin dapat melihat, menonaktifkan, atau memverifikasi akun.
  • Prioritas: Medium

- FR-031: Laporan Penjualan & Aktivitas

  • Deskripsi: Sistem menghasilkan laporan penjualan UMKM dan transaksi.
  • Prioritas: Medium

4. Kebutihan Non-Fungsional

4.1 Performance Requirements

  • Response time maksimal 3 detik.
  • Dapat menangani 100+ user bersamaan.
  • Query database dioptimasi dengan indexing.

4.2 Security Requirements

  • Password dienkripsi menggunakan bcrypt (salt 10).
  • Seluruh komunikasi melalui HTTPS/SSL.
  • JWT token untuk autentikasi session.
  • Input validation & protection dari SQL Injection & XSS.

4.3 Usability Requirements

  • Responsive design (desktop, tablet, mobile).
  • UI konsisten dengan komponen reusable.
  • Notifikasi & indikator loading.

4.4 Reliability Requirements

  • Uptime minimal 99%.
  • Backup database otomatis setiap hari.
  • Error logging & server monitoring real-time.

5. Spesifikasi Interface

5.1 User Interface

  • Halaman Beranda: Menampilkan produk UMKM, kategori, dan promo.
  • Dashboard Penjual: Statistik penjualan, daftar produk, manajemen toko.
  • Halaman Pembeli: Katalog produk, keranjang belanja, riwayat pesanan.
  • Dashboard Admin: Kontrol pengguna, laporan transaksi, dan monitoring aktivitas.

5.2 Hardware Interface

  • Kompatibel dengan perangkat desktop & mobile.
  • (Opsional) Barcode scanner untuk stok produk.

5.3 Software Interface

  • Database: MySQL 8.0
  • Payment Gateway: Midtrans / Xendit API
  • Framework: Node.js (Backend), React/Tailwind (Frontend)

5.4 Communication Interface

  • RESTful API: Format JSON
  • Methods: GET, POST, PUT, DELETE
  • Base URL: https://api.umkm-platform.id/v1

7. Constraints & Assumptions

7.1 Design Constraints

  • Kompatibel dengan browser modern.
  • Menggunakan open-source stack.
  • Database MySQL untuk efisiensi biaya.

7.2 Assumptions

  • Pengguna memiliki akses internet dan email aktif.
  • Server dengan uptime minimal 99%.
  • Dukungan maintenance selama 2 tahun.

7.3 Dependencies

  • Payment gateway & email service.
  • SSL certificate untuk keamanan data.
  • Cloud storage untuk backup produk & gambar.

8. Acceptance Criteria

  • Semua fitur fungsional (FR-001 s.d FR-031) berjalan dengan baik.
  • Tidak terdapat bug kritis yang menghambat transaksi.
  • Waktu respon < 3 detik.
  • UAT disetujui oleh stakeholder (Dinas Koperasi, UMKM, dan Admin).
  • Dokumentasi teknis dan user manual telah tersedia.
  • Pelatihan telah dilakukan untuk pengguna dan admin.
